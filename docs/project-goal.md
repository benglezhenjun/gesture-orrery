# 项目目标

## 产品目标

在 6 到 8 周内完成一个可运行的 Web 原型：用户允许摄像头访问后，系统实时识别手部动作，并用手势控制 3D 太阳系星图，包括旋转视角、缩放距离、暂停或加速时间、选择行星、切换轨道和信息层。

首版成功标准：

> 一个没有说明书的新用户，能在 1 分钟内用手势旋转、缩放、暂停时间，并选中至少一个行星查看信息。

## 当前开发目标

当前循环已经推进到 Cycle 17：手控响应和严重低 FPS 降载。

验收标准：

- 3D 场景显示太阳、八大行星、椭圆轨道、轨道倾角、小行星带、柯伊伯带、主要卫星和环系统。
- 行星有可区分的程序化纹理：撞击坑、云层、大陆、条带、大红斑、环缝或冰巨行星风暴。
- 行星按不同速度绕太阳运动，暂停和速度控制会改变模拟状态。
- 鼠标可以旋转、缩放、选择行星。
- 摄像头模块可以加载本地 MediaPipe HandLandmarker，显示视频预览和 21 点骨架。
- 双手空间控制使用锚点式直接映射：双手中心平移目标点，双手距离缩放镜头，双手连线角旋转太阳系平面。
- 双手空间控制在渲染循环中追踪最新目标，并显示场景内空间锚点环，减少识别低频带来的跳变感。
- 单手和双手输入使用自适应平滑；动作幅度大时提高响应，小幅抖动时保持稳定，双手控制还会做短时预测并设置防过冲上限。
- 手部追踪节奏会根据实际 MediaPipe 单帧延迟和当前渲染 FPS 自适应，低延迟时更快追踪，机器吃力时自动回落。
- 发送给 MediaPipe worker 的视频帧会按上一帧延迟和当前渲染 FPS 自适应缩放；延迟高或 FPS 低时降低检测尺寸，严重低 FPS 且延迟高时降到 192x108，降低模型负担。
- 双手空间控制使用更强的缩放曲线、平移比例和目标追踪速率；自动化会检查 160ms 内是否基本贴近最新目标。
- 单手和双手直接控制会按目标误差自适应提高追随速率，大幅缩放和平移时更快贴近最新手势目标。
- 双手空间控制在两次 MediaPipe 结果之间会使用短窗口预测目标，并在没有新结果时自动衰减，减少“识别帧之间停一下”的感觉。
- 双手移动到控制范围边缘时会软重锚，把当前视角作为新的抓取基线，让用户能连续放大、缩小、移动，而不是必须松手重抓。
- 单手捏合也使用锚点式直接控制：捏住时建立基线，手指开合直接缩放，手掌中心移动直接平移。
- 单手捏合在两次 MediaPipe 结果之间也会做短窗口预测，并在移动或缩放幅度较大时软重锚，减少单手缩放时必须松开重抓的情况。
- 张手、捏合、握拳会分别输出单手旋转、单手缩放/平移、暂停或恢复命令。
- 手势连续输入不通过 React 每帧重渲染，而是进入 Three.js 命令队列并由动画循环平滑消费。
- 手控模式使用动态清晰度，在帧率允许时提高画面内部分辨率，降低“为了流畅而过糊”的问题。
- 手控低 FPS 时会进入更轻的推理/预览模式：降低检测输入、拉长无手搜索间隔，并缩小摄像头预览，减少对太阳系画面的遮挡。
- 精细模型包含选中行星参考导引、表面细节线、表面微细点层、近景地貌/大气特征标签、近日点/远日点等轨道标记、轨道刻度、主要卫星标签、土星环辐条、地球城市光点、部分行星磁层/场线、命名小行星和外太阳系矮行星。
- 选中行星后相机会自动平滑靠近，让用户能直接检查该行星的近景纹理、参考导引和特征标签。
- 同一渲染帧内积压的单手/双手直接控制 move 会合并为最新目标，减少场景追随旧手势输入造成的滞后。
- 自动化测试可以证明画面非空、核心控件可用、行星选择有效、手部追踪资源可服务、fake camera 可启动、导航命令能实际改变相机和场景、模型细节结构达到门槛。

## 视觉 thesis

一个安静、精密、接近观测仪器的深空工作台，黑色空间作为主画布，太阳暖光、轨道线和状态色承担信息层级。

## 内容计划

- 主工作区：全屏 3D 太阳系。
- 顶部状态区：项目名、播放状态、FPS、时间速度。
- 右侧检查器：当前选中行星、物理信息、开发循环状态。
- 底部输入状态：当前控制源、未来手势映射、识别状态。

## 交互 thesis

- 视角旋转和缩放必须平滑，作为未来手势控制的手感基线。
- 选择行星时只做一次清晰反馈：高亮、轨道增强、信息面板同步。
- 时间控制要即时可见：暂停、恢复、速度变化都反映到行星运动和状态面板。
